# 竞品研究工作流

这份参考文件用于正式写报告前的研究阶段。

## 1. 先定义分析问题

开始检索前，先把下面四件事写清楚：

- 目标产品是谁
- 这份报告要支持什么决策
- 竞品要按什么逻辑来选
- 这次最重要的输出是战略判断、体验判断，还是商业判断

推荐把问题写成一句话，比如：

- 我想判断 DeepSeek 更应该对标哪条 Agent 路线
- 我想判断这个赛道的共性趋势和差异壁垒分别是什么
- 我想学习头部产品如何通过体验设计建立用户信任

## 2. 拉竞品大盘

大盘不要太窄，先把候选对象拉出来，再筛。

推荐至少覆盖四类：

- 直接竞品
- 邻近竞品
- 同技术路线产品
- 同商业路线产品

如果是 AI / Agent / 模型类产品，再额外看：

- 基座模型厂商
- 应用层创业公司
- IDE / 平台 / 生态入口型产品

## 3. 核心竞品筛选标准

从大盘收窄到核心竞品时，用这三个标准：

### 相关性

- 是否真的和目标问题相关
- 是否能帮助判断路线，而不只是提供热闹信息

### 代表性

- 是否代表一种独立打法
- 是否在塑造行业预期

### 非冗余性

- 放进来之后，是否真的能提供新的认知
- 如果和另一个候选产品几乎是同一逻辑，就不必都放

## 4. 信息源优先级

如果分析里包含技术判断，优先级默认如下：

1. 原始论文 / 官方技术报告 / 官方系统卡
2. 官方文档 / 发布说明 / changelog
3. 官方博客 / 官方演讲 / 官方代码仓库
4. 高质量媒体采访 / 二手分析

不要反过来。尤其不要只看社交媒体总结就下技术判断。

## 5. 时间轴字段

每个核心竞品，至少整理以下字段：

- 首次发布
- 关键功能上线
- 产品入口变化
- 价格与套餐变化
- 模型能力升级
- 长任务 / 规划 / 记忆 / 工具调用能力变化
- 平台、插件、MCP、skill、API、生态动作
- 企业级 / 权限 / 安全 / 治理动作
- 目标用户和产品叙事变化

时间轴不是做编年史，而是为了看：

- 这家公司一直在押什么
- 它能力长板放在哪里
- 它是不是在往新的目标用户扩张

## 6. 共性提取方法

整理完时间轴后，先问：

- 哪些动作是多家产品都在做的
- 哪些东西过去是亮点，现在已经接近标配
- 用户预期正在被哪些头部产品共同抬高

常见共性维度包括：

- 更联通
- 更懂环境
- 更强规划
- 更长记忆
- 更强治理

## 7. 差异提取方法

然后再问：

- 它们分别把护城河压在哪
- 它们到底想服务谁
- 它们想建立怎样的用户关系
- 它们希望占据工作流中的哪一个位置

常见差异维度包括：

- 入口与生态
- 交互表面
- 规划方式
- 留痕与治理方式
- 语气与人格感
- 商业包装方式

## 8. 三层分析问题

### 产品战略

- 核心路线是什么
- 时间轴上最关键的三次动作是什么
- 它真正想占据的是哪种入口、哪种关系、哪种工作流位置

### 产品体验

- 它是怎么建立信任的
- 它让用户觉得自己在和一个什么样的对象协作
- 它在不同阶段如何降低风险、提高掌控感

### 商业模式

- 它怎么收费
- 收费到底在卖什么
- 它的复购、留存和生态锁定逻辑是什么

## 9. 体验分析默认框架

如果没有别的更合适框架，默认按这五个阶段写：

1. 初次握手
2. 任务测试
3. 结果对焦
4. 迭代沟通
5. 长期合作

每个阶段至少回答：

- 用户此时最担心什么
- 产品如何让用户更信任它
- 产品有没有暴露自己的长板和短板
- 用户是否容易介入、校准、追溯

## 10. 研究完成检查表

开始写正文前，确认：

- 候选竞品和核心竞品已经分开
- 核心竞品的筛选理由已经写明
- 时间轴已经补齐到足以支撑判断
- 共性与差异已经拆开
- 三层分析框架已经明确
- 体验分析阶段已经定义好
